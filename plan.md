### 数学基礎解析　100問デッキ構成案（目次＆ソース資料準拠）

**1. 指数計算・指数関数・対数関数（約20問）**
第1章および第3章の内容です。計算ミスを防ぐための基礎を徹底します。
*   **指数法則と拡張**: $a^{-n} = \frac{1}{a^n}$、$a^0 = 1$、$a^{\frac{1}{2}} = \sqrt{a}$ などの基本変換。
*   **累乗根（指数根）の性質**: $\sqrt[n]{b^m} = b^{\frac{m}{n}}$ といった分数指数への変換。
*   **奇数乗根と偶数乗根の違い**: $\sqrt{(-6)^2} = 6$ だが、$\sqrt{-27} = -3$ のように、$n$乗根の$n$が奇数か偶数かによって性質が異なる点の注意。
*   **常用対数とその応用**: 桁数や最高位の数を求める問題の解法手順（※手順を穴埋めにして暗記）。

**2. 三角関数（基礎?公式?グラフ）（約40問）**
第4章の内容です。覚えるべき公式が非常に多いため、Ankiの穴埋めカードを最大限に活用します。
*   **基本性質**: $\tan\theta$ の定義域（漸近線は $\theta \neq \frac{\pi}{2} + n\pi$）や、$\sin\theta, \tan\theta$ は奇関数（原点対称）、$\cos\theta$ は偶関数（y軸対称）であること。
*   **グラフの変形**: $y = \sin(k\theta)$ の周期は $\frac{2\pi}{k}$ になるなど、グラフの拡大・縮小・平行移動のルール。
*   **公式群の穴埋め暗記**: 加法定理、倍角・半角の公式、三角関数の合成、積和・和積の公式。
    *   *作成のコツ*: 右辺をまるごと、あるいは符号や係数だけを隠すカードを作ります。

**3. 複素数とn乗根（約10問）**
第5章の内容です。指数根（n乗根）を複素数平面に拡張した内容を含めます。
*   **ド・モワブルの公式**: $(\cos\theta + i\sin\theta)^n = \cos(n\theta) + i\sin(n\theta)$ の穴埋め。
*   **複素数のn乗根**: 複素数平面上での解の配置（正n角形になる幾何学的性質など）。
*   **オイラーの公式**: $e^{i\theta} = \cos\theta + i\sin\theta$ （指数関数と三角関数の結びつき）。

**4. 微分と極限（三角関数・指数関数）（約30問）**
第6章（および数III）の内容です。計算の「型」を暗記します。
*   **極限の公式**: $\lim_{x \to 0} \frac{\sin x}{x} = 1$、$\lim_{x \to 0} \frac{\tan x}{x} = 1$。
*   **極限の計算テクニック**: 分母や分子に $1-\cos x$ がある場合、$1+\cos x$ を掛けて $\sin^2 x$ を作り出す解法パターン。
*   **導関数の公式**: $(\sin x)' = \cos x$、$(\cos x)' = -\sin x$、$(\tan x)' = \frac{1}{\cos^2 x}$。
*   **合成関数の微分**: $(\sin 2x)' = 2\cos 2x$ や $(\tan^2 x)' = \frac{2\tan x}{\cos^2 x}$ などの、置き換えを伴う微分手順。

### デッキ作成時のAnki活用ルール

この100問をAnkiで運用する際、資料に記載されている**「資格試験や高度な学習に向けたAnkiのコツ」**を導入すると、記憶の定着が飛躍的に高まります。

1.  **問題は「5秒以内で答えられる」サイズに分解する**
    「加法定理をすべて答えよ」という重いカードは作らず、$\sin(\alpha+\beta)$、$\cos(\alpha-\beta)$ など、公式1つにつき1枚のカードに分割します。ただし、各文や式は**やや大き目**（自然で読みやすい完全な表現）に保ち、極端に短く断片化しすぎないようにする。テンポよく解けることを最優先としつつ、意味の通じやすいサイズを心がける。
2.  **ショートカットで「Cloze（穴埋め）カード」を量産する**
    数式の覚えたい部分を選択し、キーボードの `Ctrl + Shift + C` (Macは `Cmd + Shift + C`) を押すだけで、簡単に穴埋め問題が作れます。
3.  **関連する論点は1つのノートにまとめる（ワントピックノート）**
    例えば「倍角の公式（$\cos 2\theta$ の3パターン）」は、1つのノート内に複数の穴埋めとしてまとめて作成します。回答時に「他のパターンはどうだったかな？」と気になった際、同じ画面ですぐに周辺知識を確認できて便利です。
4.  **MathJaxで数式を美しく表示する**
    カード内で数式を書くときは `[$]` と `[/$]` で囲むことで、きれいに表示できます（例：`[$] \lim_{x \to 0} \frac{\sin x}{x} = 1 [/$]`）。特別なインストール不要で使えるため、高校数学のデッキ作成には必須の機能です。

目次から全体像（地図）を把握し、その中の細かい公式や計算の「型」をAnkiで確実に定着させるというアプローチは、理系数学の学習において最強のメソッドになります。ぜひこの構成案をベースに作成を進めてみてください。

## Grokによる理解・解析・実装プラン・仕様定義（2026-07-07 追加）

### 1. プロジェクト全体の使命（Mission）の理解

本プロジェクト「anki-mydick-make」の核心的な目的は以下の通り：

- **学習効率化の専門家として**、Ankiを活用した「高速・高定着」の暗記システムを構築する。
- ワーキングメモリの限界を厳密に考慮し、**1問あたり5秒以内で瞬時に答えられる**範囲に収めつつ、各カード内の文や数式は**やや大き目**（自然な文や完全な式で構成）とする。極端な断片化は避ける。
- Gitと高校数学（基礎解析）の2領域について、**手作業ゼロ**でAnkiに取り込める状態まで自動化する。
- 成果物として、以下の3点を必ず生成する：
  1. `git_deck.csv`（50問・Basic形式）
  2. `math_deck.csv`（100問・Cloze + MathJax形式）
  3. `import_to_anki.py`（AnkiConnect経由の完全自動インポートスクリプト）

**【重要・新規定】カード内の文・式サイズ**
- 各デッキ内の**文や式のサイズはやや大き目**にすること。
- 極端に短く切り刻んだ断片ではなく、自然で読みやすい長さの文や、完全な数式・コマンド表現を1単位として使用する。
- 5秒ルールは厳守するが、「短すぎて意味が取りにくい」カードは避ける。

この指示書（plan.md）は、**Grok自身に対するプロンプト**としても機能しており、将来的に同じ指示を繰り返し与えても一貫した成果物が得られるように設計されている。

  - ディスプレイ（独立行）: `[$$] ... [/$$]`
- **主要トピックと配分目安**（基礎解析目次準拠）:
  - 指数計算・指数関数・対数関数：約20問
  - 三角関数（性質・グラフ・加法定理・倍角・合成など）：約40問
  - 複素数とn乗根（ド・モアブル、オイラー公式）：約10問
  - 微分と極限（三角・指数関数）：約30問
- **作成方針**:
  - 公式1つにつき基本的に1カード（または1ノートに論理的に関連する複数穴埋め）。
  - 「計算の型」「頻出の変換パターン」を優先的に穴埋め化。
  - 例: `[$]\lim_{x \to 0} \frac{\sin x}{x} = {{c1::1}}[/$]`
- **既存素材**: トピック別構成案のみ。**具体的な100問の作成が必要**。

#### 【要件3】AnkiConnect自動化スクリプト
- **言語**: Python
- **必須ライブラリ**: `requests`
- **AnkiConnect API**（localhost:8765）:
  - デッキ作成: `createDeck`
  - ノート追加: `addNote`（繰り返し実行）
  - ノートタイプ:
    - Git → `"Basic"` （将来的にImageフィールド追加も検討）

  - 重複追加防止の工夫（可能なら）
- **前提条件**: Anki本体が起動中 + AnkiConnectアドオンが有効（ポート8765）であること。
- **出力**: ユーザーが実行するだけで全カードがAnkiに取り込まれる状態。

### 4.7 設計上の注意

1. **Magit / Avy / Embark のキーはコンテキスト依存**: 表面に `【Magit】` `【Avy】` 等のプレフィックスを付け、他カテゴリの同名キー（例: `k`, `c`, `TAB`）と混同しない。
2. **Paredit の C-k** は通常の `C-k`（1行切り取り）と異なる。表面文で「S式」「括弧の構造を保持」と明示する。
3. **Org-mode のキー** はユーザー設定（`C-c a` 等）で差異がありうる。裏面に `(一般的な割当)` と注記済みのものはそのまま維持。
4. **77問の厳守**: 一次資料と生成結果の行数が一致することを `generate_emacs_deck.py` の `validate_cards()` で assert する。
5. **同一裏面キーの共存**: `C-k`（基本・編集＝行キル）と `C-k`（Paredit＝S式キル）、`k`/`c`/`TAB`（Magit/Avy/Embark）など、裏面が同じでも**表面のプレフィックスで区別**する現行設計は意図的。変更しない。
6. **上位仕様との整合**: 本要件追加によりプロジェクト総カード数は **227問**（§4.8）。旧 §6 の DoD（150問）は Emacs 追加前の記述のため、Task 5 完了時に §6 も更新する。
